#第二章线性表
1.线性表的定义：由N个类型相同的数据元素的有限序列
记作（a1, a2, a3, a4, …..an）,n是线性表长度
N=0, 称为空表
N>0, 除第一个元素无直接前驱，最后一个元素无直接后继外，其余每个元素只有一个直接前驱，和一个直接后继，数据元素之间具有一对一的关系。

2.线性表特点：
同一性：线性表由同类元素组成，每一个ai必须属于同一数据对象。
有穷性：有限个数据元素组成，表长度就是数据元素个数。
有序性：相邻数据元素之间存在着序偶关系< a i，a i+1 >

3.ADT基本操作:
initList（）初始化
destoryList（）销毁
clearList（）置空

4.顺序存储结构：

#define MAXSIZE 100
typedef int ElemType;
typedef struct node
{
ElemType elem[MAXSIZE];
int last;
}seqList;

5.线性表两种基本的查找运算
（1）按序号查找 getData（L，i）
（2）按内容查找 locate（L., e）
6.插入操作：定位，后移，插入
7.删除操作：定位
8.顺序表合并算法：LA，LB为两个递增有序排列。将其合并成一个递增有序的顺序表C
算法思路：
（1）初始化：LC为空表
（2）比较循环：LA.[i]，LB[i]，较小的放入LC中，比较完指针后移一位
（3）将剩余表循环复制到LC表中。

#第一章绪论
1.数据结构的定义和作用？
（1）数据：描述客观事物的数值，字符以及能输入机器且能被处理的而各种符号的集合。
（2）数据元素：组成数据的基本单位，是数据集合的个体。在计算机中通常作为一个整体进行考虑和处理。表的一行。
（3）数据项：表的一列。 ps：见图一
（4）数据对象：性质相同的数据元素的集合，数据的子集。
（5）数据结构：相互之间存在一种或多种特定关系的数据元素的集合，他指的是数据元素之间的相互关系，即数据的组织形式。（eg：表结构，树结构，图结构）

（6）数据类型：性质相同的值集合以及定义在这个值集合上的一组操作的总称。
（7）抽象的数据类型：ADT，定义了一个数据对象，数据对象中各元素之间的结构关系，以及一组处理数据的操作。

2.数据结构研究什么？
（1）逻辑结构：线性/非线性。数据元素间逻辑关系的描述。
（2）四类基本数据结构：集合（属于），线性（1对1），树形（1对多），网状（多对多）。
（3）存储结构：物理结构，顺序/非顺序。逻辑结构在计算机中存储映像（内存地址等）
（4）运算集合：ps：见图二
（5）算法特性：有限，确定，输入，输出，可行。
（6）算法设计要求：正确，可读，健壮，高效。
（7）性能评价：效率与问题规模N有关
（8）语句频度：语句在算法中重复执行的次数
（9）时间复杂度：以语句频度刻画随问题规模N增加的函数的执行时间量度

30天零基础学绘画
第00课 前言

绘画九大基本法则：
1.压缩：扭曲物体，以产生某部分比较接近你眼睛的错觉。
2.位置：把物体置于画面较低的位置，以表示物体比较接近你的眼睛。
3.尺寸：把物体画的大一些，以表示物体比较接近你的眼睛。
4.重叠：把一个物体画在另一个物体前面，以产生它比较接近你眼睛的错觉。
5.阴影：把一个物体背光源部分画的深些，以产生深度的错觉。
6.投影：把一个物体背光源近处的地面画的深些，以产生深度的错觉
7.轮廓线：画出一个包住物体的曲线，给其体积感和深度感。
8.水平线：画出一条水平参考线，以区分画面中不同物体与你视线之间的距离。
9.密度：把物体画的亮些，细节少一点，以产生距离上的视错觉。


第01课 球形
第一步，画一个圆
第二步，确定你要的光源位置
第三步，画出投影（背着光源且紧连物体）

第四步，在背着光源的部分涂上阴影

注：在离光源最远的圆的边缘处涂得更深些，。在暗部向亮部过渡的地方啊将其涂得浅一点，这叫做调和阴影。这是让平面物体在纸上“立”起来的有效方法。

第一课课后作业：画一个苹果，光源在右面较低处的位置
